# v1.2 Trigger Core — Design Lock

Unified event-driven trigger model. All trigger sources (manual, interval, block, log, mempool, webhook) share **one schema, one MCP API, one worker trait, one sandbox extension**. v1.2 spike ships the abstraction + `manual` + `interval` workers. v1.3+ adds remaining workers without touching schema or API.

---

## Architecture

```
┌─ Trigger Sources (workers) ─────────────────────────────────────┐
│  manual  interval   block   log   mempool   webhook             │
│   (MCP)  (tokio)    (WSS)  (WSS)  (WSS)    (HTTP)               │
│     │       │         │      │      │        │                  │
│     └───────┴─────────┴──┬───┴──────┴────────┘                  │
│                          ▼                                       │
│              mpsc::Sender<TriggerEvent>                          │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─ Dispatcher ────────────────────────────────────────────────────┐
│  on event(e):                                                   │
│    1. lookup trigger by id                                      │
│    2. if disabled → skip                                        │
│    3. if predicate_js → eval (sandbox) → bool                   │
│    4. if dedup_key in trigger_events within window → skip       │
│    5. else: insert trigger_events row, fire strategy_run        │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
            existing strategy_run pipeline (UNCHANGED)
            (policy → sim → 7702 batch → broadcast → journal)
```

## Data Model — `executor-state`

### New tables (added to `SCHEMA_SQL` in `crates/executor-state/src/schema.rs`)

```sql
CREATE TABLE IF NOT EXISTS triggers (
    id              TEXT PRIMARY KEY,           -- ULID
    strategy_id     TEXT NOT NULL REFERENCES strategies(id),
    kind            TEXT NOT NULL,              -- "manual"|"interval"|"block"|"log"|"mempool"|"webhook"
    config_json     TEXT NOT NULL,              -- kind-specific JSON (interval_ms, filter, etc.)
    predicate_js    TEXT,                       -- optional (event) => bool function source
    enabled         INTEGER NOT NULL DEFAULT 1, -- 0/1 bool
    last_fired_at   TEXT,
    created_at      TEXT NOT NULL,
    dedup_window_ms INTEGER                     -- nullable; 0/null disables dedup
);
CREATE INDEX IF NOT EXISTS idx_triggers_strategy_id ON triggers(strategy_id);
CREATE INDEX IF NOT EXISTS idx_triggers_enabled_kind ON triggers(enabled, kind);

CREATE TABLE IF NOT EXISTS trigger_events (
    id              TEXT PRIMARY KEY,           -- ULID
    trigger_id      TEXT NOT NULL REFERENCES triggers(id),
    event_json      TEXT,                       -- kind-specific payload
    fired_at        TEXT NOT NULL,
    run_id          TEXT,                       -- nullable; null if predicate/dedup skipped
    dedup_key       TEXT,
    skipped_reason  TEXT                        -- "predicate_false"|"dedup"|"disabled"|"strategy_deleted"|null
);
CREATE INDEX IF NOT EXISTS idx_trigger_events_trigger_id ON trigger_events(trigger_id);
CREATE INDEX IF NOT EXISTS idx_trigger_events_dedup
    ON trigger_events(trigger_id, dedup_key, fired_at);
```

### New CRUD module — `crates/executor-state/src/triggers.rs`

Mirrors `strategies.rs` pattern. Functions on `StateStore`:

- `register_trigger(input: RegisterTriggerInput) -> Result<Trigger>` — content-addressed id by hash of `(strategy_id, kind, config_json, predicate_js)`; idempotent on same source.
- `list_triggers(filter: Option<TriggerListFilter>) -> Result<Vec<TriggerSummary>>` — filter by enabled / kind / strategy_id.
- `get_trigger(id) -> Result<Option<Trigger>>`
- `delete_trigger(id) -> Result<()>` — hard delete (no soft delete for triggers; cleaner).
- `set_trigger_enabled(id, enabled) -> Result<()>`
- `record_trigger_event(...) -> Result<TriggerEvent>`
- `list_trigger_events(trigger_id, limit) -> Result<Vec<TriggerEvent>>`
- `check_dedup(trigger_id, dedup_key, window_ms) -> Result<bool>` — true if duplicate found in window

## Runtime — Daemon Mode

### Worker trait — `crates/executor-mcp/src/triggers/mod.rs` (new module)

```rust
pub struct TriggerEvent {
    pub trigger_id: String,
    pub payload: serde_json::Value,
    pub dedup_key: Option<String>,
}

#[async_trait::async_trait]
pub trait TriggerWorker: Send + 'static {
    fn kind() -> &'static str where Self: Sized;
    async fn run(self: Box<Self>, events: mpsc::Sender<TriggerEvent>);
}
```

### Dispatcher — `crates/executor-mcp/src/triggers/dispatcher.rs`

```rust
pub struct Dispatcher {
    state: Arc<Mutex<StateStore>>,
    server: Arc<ExecutorServer>,  // to call strategy_run path
}

impl Dispatcher {
    pub async fn run(self, mut events: mpsc::Receiver<TriggerEvent>) {
        while let Some(e) = events.recv().await {
            // 1. load trigger
            // 2. enabled check
            // 3. predicate eval (Sandbox::execute with ctx.event=e.payload)
            // 4. dedup check via state.check_dedup
            // 5. fire strategy_run (same code path as MCP tool) — pass event into ctx
            // 6. record trigger_event row
        }
    }
}
```

### Workers — v1.2 ships 2

**`triggers/workers/manual.rs`** — the existing `strategy_run` MCP tool routes through dispatcher (uniform code path). Manual = no background loop; MCP tool synthesizes a TriggerEvent and pushes to channel.

**`triggers/workers/interval.rs`**:

```rust
pub struct IntervalWorker {
    pub trigger_id: String,
    pub interval_ms: u64,
}
#[async_trait::async_trait]
impl TriggerWorker for IntervalWorker {
    fn kind() -> &'static str { "interval" }
    async fn run(self: Box<Self>, events: mpsc::Sender<TriggerEvent>) {
        let mut ticker = tokio::time::interval(Duration::from_millis(self.interval_ms));
        ticker.set_missed_tick_behavior(MissedTickBehavior::Skip);
        loop {
            ticker.tick().await;
            let _ = events.send(TriggerEvent {
                trigger_id: self.trigger_id.clone(),
                payload: serde_json::json!({ "kind": "interval", "ts": now_rfc3339() }),
                dedup_key: None,
            }).await;
        }
    }
}
```

### Worker pool lifecycle

`ExecutorServer::from_config` after policy load:
1. Load all `triggers WHERE enabled = 1`.
2. For each non-manual trigger, spawn matching worker on tokio runtime.
3. Spawn dispatcher consuming the shared `mpsc::Receiver`.
4. Store `WorkerPool` handle on `ExecutorServer` for shutdown.

`trigger_register`/`trigger_enable`/`trigger_disable` mutate runtime pool live:
- enable → spawn worker
- disable / delete → abort worker

Stdio MCP server and worker pool coexist in the same process. `main.rs` keeps `serve(stdio()).await?; service.waiting().await?;` — workers run as detached tokio tasks.

## Sandbox Extension — `strategy-js`

### CtxHost trait: add `event(&self) -> Option<&serde_json::Value>` default `None`.

### Sandbox install: `ctx.event` JS-side variable

In `sandbox.rs`, after installing `ctx.actions` / `ctx.evm`, install:

```rust
if let Some(event_val) = host.event() {
    let event_js = json_to_qjs_value(c.clone(), event_val)?;
    ctx_obj.set("event", event_js)?;
}
```

If no event (e.g., manual trigger via MCP `strategy_run` tool), `ctx.event` is `null`.

### Predicate evaluator

A predicate is a JS function `(event) => bool`. Reuse `Sandbox::execute`:
- Wrap as `(() => { const __fn = (<predicate_source>); return !!__fn(__event); })()`
- Run with a minimal `CtxHost` that only carries the event (no provider, no journal — predicates are fast and read-free).
- Return value: bool.

If predicate throws / non-bool / timeout → treat as `false` (defensive), log warn, record `skipped_reason: "predicate_error"`.

### Strategy runtime change

In `tools.rs` `strategy_run` path, when called via dispatcher, the RuntimeContext is built with an additional `event` field threaded into the sandbox so `ctx.event` is visible inside the strategy.

For manual trigger (no event), `ctx.event = null`.

## Workstreams (parallel agents)

Four streams, three crates, low conflict.

### Stream A — `executor-state` + `executor-core` (DB + types)
Owner: agent A
- Add `triggers`, `trigger_events` to `SCHEMA_SQL` in `schema.rs`.
- Create `executor-state/src/triggers.rs` with the 8 CRUD functions.
- Wire `pub use` in `lib.rs`.
- Create `executor-core/src/schema/trigger.rs` with `RegisterTriggerInput`, `TriggerKind`, `Trigger`, `TriggerSummary`, `TriggerEvent`, `TriggerListFilter` types (Serialize+Deserialize+JsonSchema).
- Unit tests: idempotent register, list with filters, dedup check window.

### Stream B — `strategy-js` (sandbox extension)
Owner: agent B
- Add `event(&self) -> Option<&serde_json::Value>` to `CtxHost` trait (default None).
- In `Sandbox::execute`, after ctx.actions/evm install, install `ctx.event` if host provides one.
- Add `Sandbox::evaluate_predicate(source, event) -> Result<bool, RuntimeError>` helper.
- Update `RuntimeContext` to carry optional event + return it via `event()`.
- Update `CtxStub` to optionally carry event for tests.
- Unit tests: predicate returns true / false / non-bool / throws.

### Stream C — `executor-mcp` MCP tools
Owner: agent C
- Depends on stream A types (use them as Parameters).
- Add 7 `trigger_*` MCP tools to `tools.rs`.
- Add `trigger://{id}` and `trigger-events://{id}` resource templates to `resources.rs`.
- Update `ExecutorServer` struct + `from_config` to hold trigger CRUD handles (state already gives us this).
- Integration tests: register/list/get/delete/enable/disable round-trip via stdio.

### Stream D — `executor-mcp` daemon + workers
Owner: agent D
- Depends on stream A (state CRUD) + stream B (predicate eval).
- Create `crates/executor-mcp/src/triggers/` module: `mod.rs`, `dispatcher.rs`, `workers/manual.rs`, `workers/interval.rs`.
- Implement `TriggerWorker` trait + `WorkerPool` (handle to spawned workers).
- Implement `Dispatcher::run` (loads trigger, evals predicate, checks dedup, calls strategy_run with event).
- Wire `ExecutorServer::from_config` to: load triggers, spawn pool, spawn dispatcher.
- Wire `trigger_enable/disable` to mutate live pool.
- Integration test: register interval trigger with 100ms interval + 5 second test → expect 50 ± events fired.
