## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> `strategy_run` | Untrusted strategy source and generated actions cross into the runtime gate. |
| `strategy_run` -> policy evaluator | Normalized actions and chain metadata cross into deny-by-default access-control logic. |
| `strategy_run` -> journal storage | Gate decisions become persistent audit records used for repudiation defense. |
| `strategy_run` -> EVM simulation RPC | Only policy-approved actions may reach `eth_call`; policy-denied actions must be journaled as simulation skipped, not simulated. |
