rquickjs 0.11 ctx-injection idiom (RESEARCH §"ctx Host API" + Code Examples):

```rust
ctx.with(|c: rquickjs::Ctx<'_>| -> Result<_, RuntimeError> {
    // Build a JS object literal mirroring D-04.
    let ctx_obj = rquickjs::Object::new(c.clone())?;

    // Strategy / run identity (read-only strings).
    let strategy_obj = rquickjs::Object::new(c.clone())?;
    strategy_obj.set("id", host.strategy_id())?;
    strategy_obj.set("name", host.strategy_name())?;
    ctx_obj.set("strategy", strategy_obj)?;

    let run_obj = rquickjs::Object::new(c.clone())?;
    run_obj.set("id", host.run_id())?;
    ctx_obj.set("run", run_obj)?;

    // ctx.now() — host-bound function. The captured `now_millis()` value
    // is taken at injection time; agent-visible determinism is "now() returns
    // a snapshot at run start" — adequate for Phase 3.
    let now_value = host.now_millis() as f64;
    let now_fn = rquickjs::Function::new(c.clone(), move || -> rquickjs::Result<f64> {
        Ok(now_value)
    })?;
    ctx_obj.set("now", now_fn)?;

    // ctx.log(...) — host-bound function that captures *args into a host buffer.
    // Closure must satisfy `Fn` and `'static` — use Rc<RefCell<Vec<String>>> or
    // pass the buffer through a shared cell. Implementation detail in Task 1.
    // Plan 03-03 will wire this into RuntimeContext::append_log.

    // ctx.actions.noop() — returns "noop" literally.
    let actions_obj = rquickjs::Object::new(c.clone())?;
    let noop_fn = rquickjs::Function::new(c.clone(), || -> rquickjs::Result<String> {
        Ok("noop".to_string())
    })?;
    actions_obj.set("noop", noop_fn)?;
    ctx_obj.set("actions", actions_obj)?;

    c.globals().set("__ctx", ctx_obj)?;
    // ... continue with the (SOURCE)(__ctx) wrapping pattern from Plan 03-01.
});
```

**Pitfall 2 carry-over:** `ctx.log` MUST NOT do DB IO inside the JS callback. Buffer in a host-side `Vec<String>`; flush after `Sandbox::execute` returns control to the host. RuntimeContext::flush is the flush boundary.

**Closure lifetime constraint:** rquickjs `Function::new` typically requires the closure to be `'static + Send` (depending on rquickjs internals). This means the closure CANNOT directly borrow `&mut host`. Approach: use an `Rc<RefCell<Vec<String>>>` clone shared between `Sandbox::execute` and the closure; after `ctx.with` returns, drain the buffer and call `host.append_log(msg)` for each captured message. (Or use a thread-local. Whichever rquickjs 0.11 makes most ergonomic — verify in Task 1's first implementation pass.)

    ```rust
    /// D-12 transition guard. Atomically updates `runs.status` only when the
    /// row's current status equals `from`. Returns `StateError::InvalidInput`
    /// (NOT `NotFound`) when the row exists but is in a different state — the
    /// caller's invariant is violated, not the row's existence.
    ///
    /// `NotFound` is returned only when the row does not exist at all.
    /// Reserved-variant gate (`phase2_emittable`) is enforced for both `from`
    /// and `to` (you cannot transition INTO a reserved variant from Phase 3 code,
    /// nor declare you expect a reserved one).
    ///
    /// Closes 02-REVIEW MR-01: prior `update_run_status` was unconditional and
    /// could overwrite a terminal status with an earlier-stage status. Phase 3's
    /// `strategy_run` handler MUST use this transition-guarded API for every
    /// status change.
    pub(crate) fn update_run_status_with_transition(
        conn: &Connection,
        run_id: &str,
        from: RunStatus,
        to: RunStatus,
    ) -> Result<(), StateError> {
        if !from.phase2_emittable() || !to.phase2_emittable() {
            return Err(StateError::InvalidInput(format!(
                "transition {from:?} → {to:?} involves a Phase 5/6 reserved status; \
                 not allowed from Phase 3 code paths"
            )));
        }
        let finished_at = matches!(to, RunStatus::Succeeded | RunStatus::Failed)
            .then(super::strategies::now_rfc3339);
        let affected = conn.execute(
            "UPDATE runs SET status = ?1, finished_at = COALESCE(?2, finished_at) \
             WHERE id = ?3 AND status = ?4",
            params![status_to_wire(to), finished_at, run_id, status_to_wire(from)],
        )?;
        if affected == 0 {
            // Distinguish NotFound vs InvalidInput by re-querying the row.
            let exists: bool = conn
                .query_row(
                    "SELECT 1 FROM runs WHERE id = ?1",
                    params![run_id],
                    |_| Ok(()),
                )
                .optional()?
                .is_some();
            if !exists {
                return Err(StateError::NotFound(format!("run {run_id}")));
            }
            return Err(StateError::InvalidInput(format!(
                "run {run_id} not in expected state {from:?} (transition guard)"
            )));
        }
        Ok(())
    }

            .map_err(|e| classify_qjs_error(e, &timed_out))?;

        // deterministic-per-run (snapshot semantics); Phase-4+ may revisit if
        // the agent needs intra-strategy clock progression.
        let now_value = host.now_millis() as f64;

            Ok(now_value)
        }).map_err(|e| classify_qjs_error(e, &timed_out))?;
        ctx_obj.set("now", now_fn)
            .map_err(|e| classify_qjs_error(e, &timed_out))?;

        // Args coerced to string via JS-side String(...) before reaching us.
        // We accept a Rest-Args of rquickjs::Value and stringify per the JS
        // String() rules.
        //
        // Implementation note: rquickjs 0.11's Function::new closure bounds
        // determine whether we can capture `Rc<RefCell>` directly. If the
        // closure must be `'static + Send`, use Rc (single-threaded — runs
        // entirely inside one spawn_blocking worker, so Send is not required
        // in practice; rquickjs may still demand it). Worst case: use a
        // thread-local or a static `OnceCell`.
        let buf = log_buffer.clone();
        let log_fn = rquickjs::Function::new(c.clone(), move |args: rquickjs::Rest<rquickjs::Value>| -> rquickjs::Result<()> {
            let parts: Vec<String> = args.0.iter().map(|v| qjs_value_to_display_string(v)).collect();
            buf.borrow_mut().push(parts.join(" "));
            Ok(())
        }).map_err(|e| classify_qjs_error(e, &timed_out))?;
        ctx_obj.set("log", log_fn)
            .map_err(|e| classify_qjs_error(e, &timed_out))?;

        // ctx.actions.noop() — returns the literal string "noop".
        let actions_obj = rquickjs::Object::new(c.clone())
            .map_err(|e| classify_qjs_error(e, &timed_out))?;
        let noop_fn = rquickjs::Function::new(c.clone(), || -> rquickjs::Result<String> {
            Ok("noop".to_string())
        }).map_err(|e| classify_qjs_error(e, &timed_out))?;
        actions_obj.set("noop", noop_fn)
            .map_err(|e| classify_qjs_error(e, &timed_out))?;
        ctx_obj.set("actions", actions_obj)
            .map_err(|e| classify_qjs_error(e, &timed_out))?;

        c.globals().set("__ctx", ctx_obj)
            .map_err(|e| classify_qjs_error(e, &timed_out))?;

        // ... continue with the IIFE wrapping + eval pattern from Plan 03-01.
    });

    // After ctx.with returns: drain the log buffer into the host.
    if let Ok(mut buf) = Rc::try_unwrap(log_buffer).map(|cell| cell.into_inner()) {
        for msg in buf.drain(..) {
            host.append_log(msg);
        }
    } else {
        // Some closure clone is still alive — should not happen in practice.
        // Be defensive: borrow_mut() and drain.
        // (Implementation detail; executor confirms via test.)
    }

    //! `RuntimeContext` — production [`CtxHost`] impl backed by an
    //! `Arc<tokio::sync::Mutex<StateStore>>`. Buffers logs during JS
    //! execution; `flush` drains them to `journal_logs` and emits the
    //! Phase-3 source-read marker (STJ-03) in a single mutex acquisition.
    //!
    //! Plan 03-03's `strategy_run` handler is the single caller. It runs
    //! `Sandbox::execute(source, &mut runtime_ctx)` inside a
    //! `tokio::task::spawn_blocking` block (rquickjs `Runtime` is `!Sync`)
    //! and then invokes `runtime_ctx.flush(&mut state)` in a separate
    //! `spawn_blocking` block.
    //!
    //! This module does NOT touch `runs.status` — D-12 transitions are the
    //! handler's responsibility. RuntimeContext only owns ctx surface +
    //! source-read marker + log flush.

    use std::sync::Arc;
    use tokio::sync::Mutex;

    use executor_state::{StateError, StateStore};

    use crate::sandbox::CtxHost;

    /// Snapshot-of-now provider — Phase 3 default uses `chrono::Utc::now`,
    /// tests inject a fixed value for determinism. Boxed to avoid leaking the
    /// generic into `RuntimeContext`'s public type.
    pub type NowMillisProvider = Arc<dyn Fn() -> i64 + Send + Sync>;

    pub struct RuntimeContext {
        state: Arc<Mutex<StateStore>>,
        strategy_id: String,
        strategy_name: String,
        run_id: String,
        now_provider: NowMillisProvider,
        log_buffer: Vec<String>,
        source_read_pending: bool,    // set true on construction; cleared after flush
    }

    impl RuntimeContext {
        pub fn new(
            state: Arc<Mutex<StateStore>>,
            strategy_id: String,
            strategy_name: String,
            run_id: String,
            now_provider: NowMillisProvider,
        ) -> Self {
            Self {
                state,
                strategy_id,
                strategy_name,
                run_id,
                now_provider,
                log_buffer: Vec::new(),
                source_read_pending: true,
            }
        }

        /// Default chrono-backed clock provider (Phase 3 v1 — D-04).
        pub fn default_clock() -> NowMillisProvider {
            Arc::new(|| chrono::Utc::now().timestamp_millis())
        }

        /// Drain the buffered logs and the source-read marker into the
        /// `StateStore`. Called from the strategy_run handler AFTER
        /// `Sandbox::execute` returns control. Caller MUST run this inside
        /// `tokio::task::spawn_blocking` because `state.blocking_lock()` is
        /// invoked here.
        ///
        /// Idempotent: a second call after a successful flush writes zero
        /// new rows (source_read_pending is cleared, log_buffer is drained).
        pub fn flush(&mut self) -> Result<(), StateError> {
            let mut store = self.state.blocking_lock();
            // 1. Source-read marker (STJ-03).
            if self.source_read_pending {
                store.record_source_read(
                    &self.run_id,
                    "strategy_source",
                    &self.strategy_id,
                    None,
                )?;
                self.source_read_pending = false;
            }
            // 2. Logs (D-04 ctx.log → journal_logs).
            for msg in self.log_buffer.drain(..) {
                store.record_log(&self.run_id, &msg)?;
            }
            Ok(())
        }
    }

    impl CtxHost for RuntimeContext {
        fn strategy_id(&self) -> &str { &self.strategy_id }
        fn strategy_name(&self) -> &str { &self.strategy_name }
        fn run_id(&self) -> &str { &self.run_id }
        fn now_millis(&self) -> i64 { (self.now_provider)() }
        fn append_log(&mut self, message: String) {
            self.log_buffer.push(message);
        }
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Strategy JS source → host process via ctx surface | Strategies can read `ctx.strategy.id` / `.name` / `ctx.run.id` (read-only strings the host injected). Cannot mutate them — the JS object is constructed inside `ctx.with` and the strategy receives a reference; setting `ctx.strategy.id = "X"` modifies the local view but the host never re-reads from the JS object. |
| `ctx.log` callback → host log_buffer | rquickjs invokes the host-bound `Function` callback synchronously inside JS execution. The callback appends to a `Rc<RefCell<Vec<String>>>` (or equivalent shared cell) — no DB IO, no async, bounded memory (each log is bounded by `MEMORY_LIMIT_BYTES`). |
| `RuntimeContext::flush` → `StateStore` mutex | flush() acquires `state.blocking_lock()` ONCE; writes all source_read + log rows under that one acquisition; releases. Callers run flush inside `spawn_blocking` (Phase-2 carry-over invariant). |
| `update_run_status_with_transition` → runs.status column | Atomic `UPDATE ... WHERE id=? AND status=?from`; returns `InvalidInput` on transition mismatch (closes 02-REVIEW MR-01 silent-overwrite race). |
