<anti_patterns>
## Anti-patterns to avoid (from Phase 2 deviations + RESEARCH pitfalls)

1. **Do not enable rquickjs `loader` / `dyn-load` / `futures` / `parallel` features.** D-01 / Cargo audit asserts none are pulled. Enabling any would either compromise the sandbox (`loader`/`dyn-load`) or change the threading characteristics (`parallel`).
2. **Do not use `Context::full`.** It enables `module` / `import` / `require` intrinsics. Use `Context::base` exclusively. Acceptance grep asserts zero matches.
3. **Do not call `set_memory_limit(0)`.** Pitfall 3 — that means UNLIMITED in rquickjs. Use the `MEMORY_LIMIT_BYTES` constant directly. `limits::tests` regression-guards this.
4. **Do not return `rquickjs::Value` past `Context::with`.** Pitfall 5 — Values are `'js`-bound and become invalid outside the closure. Convert to `serde_json::Value` INSIDE the closure.
5. **Do not store `Runtime` in a long-lived `Arc<Mutex<>>`.** Pitfall 6 — `!Sync` without `parallel`. Construct fresh per call inside `spawn_blocking` (caller's responsibility — `Sandbox::execute` is synchronous so the trap is one level up at Plan 03-03).
6. **Do not pool runtimes in v1.** RESEARCH Q6 — fresh per call. If startup cost is observed > 50 ms in production, pool BEHIND the API in a future phase.
7. **Do not write to stdout / stderr inside the sandbox crate.** Per-crate `#![deny(clippy::print_stdout, clippy::print_stderr, clippy::dbg_macro)]` in `lib.rs`. `tracing::*` for any logs.
8. **Do not promote `rquickjs` to `[workspace.dependencies]`.** Phase-2 precedent — only `executor-core` is shared; everything else is per-crate. Comment block in `Cargo.toml` documents the rule.
9. **Do not introduce a separate error enum per module.** `executor-state` deliberately uses one `StateError`. Same here: one `RuntimeError` for `strategy-js`. (PATTERNS §6 carry-over.)
10. **Do not assert on rquickjs error message strings without a fallback.** `classify_qjs_error` should accept the message and fall back to `Exception(msg)` if no heuristic matches; do NOT panic on unrecognised messages.
11. **Do not skip the `cargo tree` audit.** It is the only durable guarantee that future dependency churn doesn't accidentally enable a forbidden feature.
12. **Do not delete D-11 regression tests if rquickjs's intrinsic surface changes.** If a rquickjs upgrade adds a new intrinsic, the test name list MUST be expanded to cover it explicitly.
</anti_patterns>
