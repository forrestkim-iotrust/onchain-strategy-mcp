<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → execution_get/resource URI | Untrusted run ID input crosses MCP boundary. |
| SQLite execution rows → MCP response | Persisted execution facts are serialized for agents. |
| execution report → operator/agent | Report may include signer address and tx hashes but must not include private-key material. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Tampering | `execution_get` input | mitigate | Treat `execution_id` as run ID, validate resource URI shape, and use typed store lookup; unknown IDs return not_found. |
| T-06-03-02 | Repudiation | `build_execution_report` | mitigate | Build reports only from persisted run and execution rows, not transient memory or journal-only summaries. |
| T-06-03-03 | Information Disclosure | `ExecutionGetResponse` | mitigate | Include signer address/tx hashes/receipt data only; never include env-var private-key value or raw signer debug output. |
| T-06-03-04 | Information Disclosure | resource errors | mitigate | Return stable not_found/malformed_id envelopes; do not expose raw SQLite or provider details in resource errors. |
</threat_model>
