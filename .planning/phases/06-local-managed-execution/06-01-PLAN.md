<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config file → server config | Untrusted local TOML controls signer env-var name and timeout. |
| OS environment → signer boundary | Private-key value crosses from process environment into signer memory. |
| signer boundary → MCP/status/journal | Only signer address and stable non-secret errors may leave signer boundary. |
| strategy JS → host runtime | Strategy code must not access signer config, env vars, or signer handle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Information Disclosure | `executor-signer::local` | mitigate | Store only env-var name in config; never serialize private-key value; `SignerError` variants include env-var name but not env-var value. |
| T-06-01-02 | Elevation of Privilege | `Config::signer_config` | mitigate | No production private-key default; absent `[signer].private_key_env` produces `None` and downstream signing fails closed. |
| T-06-01-03 | Tampering | `LocalSignerHandle::from_env` | mitigate | Parse key with Alloy `PrivateKeySigner` and set signer chain id via `with_chain_id(Some(chain_id))`; no custom cryptography. |
| T-06-01-04 | Information Disclosure | `ExecutorServer` / strategy JS | mitigate | Store signer config only in server; do not inject signer or env into `RuntimeContext` or strategy globals. |
</threat_model>
