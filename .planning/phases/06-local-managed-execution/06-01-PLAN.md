Revised integration contract for Plan 06-02:
```rust
// Do this at the actual signing/execution boundary inside strategy_run or a helper it calls:
let signer_config = crate::config::load()
    .map_err(|e| /* stable runtime config error with run_id */)?
    .signer_config()
    .map_err(|e| /* stable signer config error with run_id */)?;

// Then fail closed when signer_config is None, before any signing/broadcast attempt.
```

This contract derives signer config from the existing config loader at the boundary where approved transactions are about to sign. It does not add `signer_config` to `ExecutorServer`, does not thread a new field through `new_with_config` / `new_with_full_config`, and does not read `EXECUTOR_PRIVATE_KEY` during server boot.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config file → config parser | Untrusted local TOML controls signer env-var name and timeout. |
| OS environment → signer boundary | Private-key value crosses from process environment into signer memory only when `LocalSignerHandle::from_env` is called. |
| signer boundary → MCP/status/journal | Only signer address and stable non-secret errors may leave signer boundary. |
| strategy JS → host runtime | Strategy code must not access signer config, env vars, or signer handle. |
| config parser → strategy_run execution boundary | `Config::signer_config` passes only env-var name and timeout; no secret value or signer handle is stored on `ExecutorServer`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Information Disclosure | `executor-signer::local` | mitigate | Store only env-var name in config; never serialize private-key value; `SignerError` variants include env-var name but not env-var value. |
| T-06-01-02 | Elevation of Privilege | `Config::signer_config` | mitigate | No production private-key default; absent `[signer].private_key_env` produces `None` and downstream signing fails closed before signing. |
| T-06-01-03 | Tampering | `LocalSignerHandle::from_env` | mitigate | Parse key with Alloy `PrivateKeySigner` and set signer chain id via `with_chain_id(Some(chain_id))`; no custom cryptography. |
| T-06-01-04 | Information Disclosure | `strategy_run` execution boundary / strategy JS | mitigate | Plan 06-02 must resolve signer config inside host runtime only; do not inject signer, env vars, or `LocalSignerHandle` into `RuntimeContext` or strategy globals. |
| T-06-01-05 | Denial of Service / Change Risk | `ExecutorServer::new_with_config`, `ExecutorServer::new_with_full_config` | mitigate | Do not edit these CRITICAL-risk constructors in this plan; if a constructor touch becomes unavoidable, isolate it in a separate checkpointed plan after a fresh GitNexus blast-radius report. |
</threat_model>
