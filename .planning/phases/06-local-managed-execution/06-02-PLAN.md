<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Phase 5 gates → signer loop | Only policy-approved and simulation-passing tx requests may reach signer. |
| signer loop → configured RPC | Locally signed transactions leave process via configured RPC URL. |
| RPC receipt → state store | Untrusted node response becomes persisted status summary. |
| state store → MCP errors/status | Stable machine-readable execution data is exposed to agents. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Elevation of Privilege | `strategy_run` execution insertion | mitigate | Call `execute_approved_actions` only after all policy and simulation rows pass; no signer call on policy/simulation denial paths. |
| T-06-02-02 | Tampering | Multi-action execution loop | mitigate | Iterate original `normalized.iter().enumerate()`, persist original `action_index`, and wait for each receipt before broadcasting the next action. |
| T-06-02-03 | Repudiation | `execution_actions` | mitigate | Persist tx hash immediately after broadcast and update same `(run_id, action_index)` row with receipt/gas/error outcome. |
| T-06-02-04 | Information Disclosure | Broadcast/receipt errors | mitigate | Map raw signer/RPC errors to stable `error_kind` strings; do not store raw private keys or unstable provider details in execution rows. |
| T-06-02-05 | Denial of Service | Receipt wait | mitigate | Use configured timeout and mark failed on timeout; no indefinite waits and no retry/replacement loop. |
</threat_model>
