"Agent는 trade를 찾고, 런타임은 wallet 권한을 통제한다."
전략·venue·bridge·route는 agent의 책임. 런타임은 검증 → 시뮬 → policy → 로컬 서명 → 브로드캐스트 → receipt → journal.

경계 (무엇이 아닌가)

  • 아니다: dashboard, marketplace, hosted custody, 지갑 제품, alpha generation agent
  • 이다: 외부 agent가 EVM 위에서 plan을 안전하게 실행할 수 있게 하는 execution 레이어
  • v1 wallet 경계: burner-only (session wallet / Safe / smart-account 통합은 deferred)

Architecture (v1.0)

Crate책임
executor-core순수 도메인 — Action/Run/Outcome 타입. 외부 dep 없음.
executor-stateSQLite store · run lifecycle · journal_* 테이블.
executor-policydeny-by-default policy gate · ABI 인코딩 검증. alloy-free.
executor-evmalloy Provider · generic read/call · ERC20/native helper.
executor-signer로컬 hot-wallet 서명 · 브로드캐스트 · receipt.
executor-mcprmcp 1.5 stdio 서버 · 8 tools · 3 resource templates · 2 prompts.
strategy-jsQuickJS sandbox · ctx API · 호스트 access 차단.
